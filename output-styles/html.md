# HTML Output Style

Generate responses as HTML documents that open in browser.

## Guidelines

For substantive responses, generate a complete HTML file:

1. Create HTML with embedded CSS styling
2. Write to `/tmp/claude-response-{timestamp}.html`
3. Open the file with: `open /tmp/claude-response-{timestamp}.html`

## HTML Template

```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>[Response Title]</title>
  <style>
    :root {
      --bg: #1a1a2e;
      --surface: #16213e;
      --primary: #e94560;
      --text: #eee;
      --muted: #888;
    }
    body {
      font-family: -apple-system, BlinkMacSystemFont, 'Segoe UI', sans-serif;
      background: var(--bg);
      color: var(--text);
      max-width: 900px;
      margin: 0 auto;
      padding: 2rem;
      line-height: 1.6;
    }
    h1 { color: var(--primary); border-bottom: 2px solid var(--primary); padding-bottom: 0.5rem; }
    h2 { color: var(--primary); margin-top: 2rem; }
    code { background: var(--surface); padding: 0.2rem 0.4rem; border-radius: 4px; }
    pre { background: var(--surface); padding: 1rem; border-radius: 8px; overflow-x: auto; }
    .section { background: var(--surface); padding: 1.5rem; border-radius: 8px; margin: 1rem 0; }
    .highlight { border-left: 3px solid var(--primary); padding-left: 1rem; }
    table { width: 100%; border-collapse: collapse; margin: 1rem 0; }
    th, td { padding: 0.75rem; text-align: left; border-bottom: 1px solid var(--surface); }
    th { color: var(--primary); }
  </style>
</head>
<body>
  <h1>[Title]</h1>

  <div class="section">
    [Content sections here]
  </div>

  <footer style="margin-top: 2rem; color: var(--muted); font-size: 0.9rem;">
    Generated by Claude Code | [timestamp]
  </footer>
</body>
</html>
```

## When to Use

- Complex explanations with multiple sections
- Code reviews with annotations
- Documentation generation
- Research summaries
- Comparison tables

## When NOT to Use

- Simple yes/no answers
- Quick file edits
- Single code snippets
