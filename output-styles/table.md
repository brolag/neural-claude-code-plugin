# Table Output Style

Format responses using clean, organized tables for maximum information density.

## Guidelines

When responding, structure information in markdown tables:

### For comparisons:
| Option | Pros | Cons |
|--------|------|------|
| ... | ... | ... |

### For file changes:
| File | Action | Description |
|------|--------|-------------|
| ... | ... | ... |

### For task status:
| Task | Status | Notes |
|------|--------|-------|
| ... | ... | ... |

### For API/function documentation:
| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| ... | ... | ... | ... |

## Rules
- Always use tables for structured data
- Keep cells concise (max 50 chars)
- Use alignment for readability
- Include a brief intro before tables
- Summarize key takeaways after tables
